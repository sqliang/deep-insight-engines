# 项目全局目录树模板指南

> 💡 阅前提示
>
> 本目录树是**初步推测**出的初探草稿，模块职责、依赖关系、调用链路以后续 架构拓扑梳理 为准。

## 目录树格式规范

### 1. 必须包裹在代码块中

**整个目录树必须放在 ` ```text ` 代码块中**，示例：

```text
repo-name/
├── src/                    # 【核心基建】源代码主目录
│   ├── main.ts            # 【核心基建】CLI 入口启动文件
│   └── utils/             # 【工具集】通用工具函数
└── README.md               # 【配置】项目说明文档
```

### 2. 注释格式规范

- **注释与目录/文件名之间保留至少 2 个空格**
- 使用统一的注释标签，标签用【】包裹
- 标签后空一格再写注释内容

### 3. 注释标签说明

| 标签 | 含义 | 视觉效果（Markdown） |
|------|------|---------------------|
| 【核心基建】 | 底层框架、核心逻辑、公共能力 | **加粗** |
| 【业务模块】 | 具体业务功能实现 | 正常 |
| 【UI 视图】 | 页面、组件、渲染相关 | 正常 |
| 【工具集】 | 通用工具函数、辅助类 | *斜体* |
| 【配置】 | 构建、环境、项目配置 | 正常 |
| 【⚠️ 待分析】 | 命名模糊 / 职责不明 / 需重点阅读 | ⚠️ 橙色提示 |
| 【废弃】 | 不再维护、遗留目录 | ~~删除线~~ |

### 4. 完整示例

```text
opencli/
├── src/                    # 【核心基建】源代码主目录
│   ├── browser/           # 【核心基建】浏览器控制模块（CDP协议、页面管理等）
│   │   ├── index.ts       # 【核心基建】模块导出入口
│   │   ├── bridge.ts      # 【核心基建】BrowserBridge - 浏览器桥接
│   │   ├── cdp.ts         # 【核心基建】CDPBridge - Chrome DevTools Protocol 实现
│   │   ├── page.ts        # 【核心基建】Page - 页面抽象接口实现
│   │   └── stealth.ts     # 【核心基建】反检测/隐身脚本
│   ├── pipeline/          # 【核心基建】管道执行引擎
│   │   ├── index.ts       # 【核心基建】模块导出入口
│   │   ├── executor.ts    # 【核心基建】管道执行器
│   │   └── steps/        # 【核心基建】管道步骤实现
│   ├── commands/          # 【业务模块】内置命令实现
│   ├── download/          # 【业务模块】下载功能
│   ├── main.ts            # 【核心基建】CLI 入口文件
│   ├── cli.ts             # 【核心基建】CLI 命令注册与 Commander 集成
│   └── ...                # 其他模块
├── clis/                  # 【业务模块】网站适配器目录（70+ 网站）
│   ├── _shared/           # 【工具集】适配器共享代码
│   ├── bilibili/          # 【业务模块】Bilibili 适配器
│   └── ...                # 其他 70+ 网站适配器
├── extension/             # 【核心基建】浏览器扩展（Browser Bridge）
├── skills/                # 【工具集】AI 技能（Claude Code / Cursor 技能）
├── docs/                  # 【配置】项目文档
├── tests/                 # 【质量保证】测试文件
├── scripts/               # 【工具集】构建和部署脚本
├── autoresearch/          # 【⚠️ 待分析】自动化研究相关（评估用）
├── .github/               # 【配置】CI/CD 配置
├── package.json           # 【配置】项目配置
├── README.md              # 【配置】英文 README
└── README.zh-CN.md        # 【配置】中文 README
```

## 标注规范（后续统一使用）

- **【核心基建】**：底层框架、核心逻辑、公共能力
- **【业务模块】**：具体业务功能实现
- **【UI 视图】**：页面、组件、渲染相关
- **【工具集】**：通用工具函数、辅助类
- **【配置】**：构建、环境、项目配置
- **【⚠️ 待分析】**：命名模糊 / 职责不明 / 需重点阅读
- **【废弃】**：不再维护、遗留目录
