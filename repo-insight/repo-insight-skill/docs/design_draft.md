
# 需求说明

专门用于深度分析和解读本地或 GitHub 源代码仓库的 Agent Skill。在 Claude Code 中全自动运行：

1. 调用后无需中间交互，直接完成所有分析工作
2. 根据需要在部分环节使用 shell 或 python 脚本来
3. Claude Code 通过工具调用进行智能分析和内容生成

## 使用方式

```
"帮我分析这个 GitHub 仓库：https://github.com/..."

或

"分析当前目录的代码"
```

## 输出结构

```shell
# 当使用 URL 模式时，代码库克隆到固定的 github-sources 目录 或用户指定目录

    # 阶段1：
    │   ├── repo_overview.md
```

# 2. 核心工作流设计
================================================================================
                      代码库深度解读核心业务工作流设计
================================================================================
【阶段一：全局扫描、输入与信息聚合、产品降维与文档基建】
   |
   +-- [输入] : 目标 GitHub 仓库地址 / 原始代码文件集合
   |
   +-- [处理] :
   |      1. 物理层过滤干扰项：排除第三方依赖包、构建产物、环境配置文件等噪音。
   |      2. 结构展平与草稿目录树生成：
   |         - 提取全局文件清单，构建基础的"全局目录层级树"。
   |         - 不仅仅是打印 Tree，而是基于 README 描述或业界通用命名规范，对目录树进行"职责域初步推测标注"，对无法推测的目录打上"待定" Tag。
   |      3. 产品视野的业务心智初始化与选型提取：
   |         - 锁定线索：优先扫描 `README.md`、`docs/` 目录、`package.json` 等全局信息。
   |         - 语义提炼：分析该仓库的核心定位、解决的具体痛点、适用的业务场景，并提取/推导典型的使用 Case。
   |         - 选型反推与对比：初步提取其核心技术方案，并分析选择该方案的背景与合理性，且强推导"方案选型对比"（即：为何采用此架构/技术，放弃了哪些替代方案，权衡点是什么）。
   |      4. 国际化基建 (Doc Translation)：
   |         - 语言嗅探：检测 README 和 docs 等目录下的核心文档语言。
   |         - 本地化生成：若非中文且无中文版本，触发翻译流，在同级目录生成如 README_zh-CN.md 等对应中文文档，不覆盖原文件。
   |
   +-- [输出] :
   |      1. 含有新增中文翻译文档的增强版目标源码集合。
   |      2. [新增] `reports/draft_tree.md`: 全局目录树，包含注释草稿，标注职责推测，作为后续阶段的"先验指导上下文"，且可被下个阶段修改
   |      3. [新增] `reports/repo_overview.md`：结构化文档 1，包含：定位/问题/场景/Case/方案初探等，能作为后续阶段的"先验指导上下文"，详细模板可参考 reference。

						   			||
                               \/

【阶段二：宏观视野、架构设计梳理与靶向拓扑】
   |
   +-- [输入] :  1. 目标源码仓库集合 (增强版，带翻译文档)
   |            2. `reports/draft_tree.md` (全局物理目录与职责推测草稿)
   |            3. `reports/repo_overview.md` (产品与业务上下文解析文档)
   |
   +-- [处理] :  1. 靶向寻路与入口解析：
			     - 读取 `reports/draft_tree.md`，提取并解析高价值入口文件（如 package.json、全局 main/index 文件）和打着"待定"标签的神秘目录。
			     - 重点分析源代码文件，定位系统的全局入口、核心路由分发点、主控中心，可结合 `reports/repo_overview.md` 的业务场景，带着"业务目的"寻找核心节点。
                2. 实体识别与拓扑推导：
				- 顺藤摸瓜，扫描入口文件引用的核心模块，梳理模块间的 Import/Export 依赖链路，以及模块 A 是如何调用或依赖模块 B 的。
				- 运用领域驱动设计 (DDD) 视角，识别代码中的核心业务实体（如 User, Agent, Order 等）及其在模块间的流转引用。
			    3. 架构梳理逆向绘制：
				 - 结合技术栈、源代码分析、模块依赖与上下文文档，综合技术栈与模块依赖，逆向推导整体架构设计与核心模块设计。
			     - 转换输出格式：将系统整体架构、路由调用关系、实体依赖、核心模块设计(3~5个)等转化为人类易读的 plainText (ASCII 字符画) 图表及较为详尽的文字说明或表格整理。
			    4. 核心数据流转时序提取 (Data Flow Tracing)：
			     - 追踪核心场景下的数据是如何从入口输入，经过哪些模块/中间件/拦截器/服务等处理，最终落盘或返回的。提取数据流各节点的交互时序与条件分支。
			     - 转换输出格式：转换为人类易懂的 mermaid、 plainText (ASCII 字符画) 图表，以及较为详尽的文字说明或表格整理。
				5. 规范化契约提取 (Contract Specification)：
				  - 扫描对外暴露的 Controller/Router 或公共暴露的 API 函数。
				  - 将提取的参数、返回值、状态码转化为 OpenSpec/OpenAPI 规范标准，内部调用逻辑转化为标准 TypeScript Interface。
   |            6. 实体树收敛：根据源码实际验证结果，推测不同目录对应的业务职责（哪些是核心业务，哪些是通用工具），修正阶段一的目录职责推测，打平未知目录。
   +-- [输出] :
			   1. [新增] `reports/architecture.md` : 《架构与拓扑梳理文档》，结构化文档2，模板参考 reference。
			   2. [新增] `reports/final_tree.md` : 经源码验证的最终版目录树


                               ||
                               \/

【阶段三：微观深度阅读与知识资产化】
   |
   +-- [输入] :
		    1. 阶段一产物: `reports/repo_overview.md`
		    2. 阶段二的所有产物:  `reports/architecture.md`、 `reports/final_tree.md`
   |        3. 目标源码仓库集合 (增强版，带翻译文档)
   |
   +-- [处理] :
   |      1. 分块调度：基于 `reports/final_tree.md`，将高复杂度模块分发给不同的子 Agent 节点（或放入队列）。
   |      2. 靶向阅读：从核心入口开始逐级阅读。从核心入口和、高复杂度/高价值的文件、难点代码开始逐级阅读
   |      3. 逻辑反推与业务对齐：阅读源码时，不仅解释技术逻辑，还要与阶段一中的`reports/repo_overview.md`  "业务场景"对齐，推导其"解决什么问题"以及"为什么这么写"（例如注释：这里的容错处理是为了应对阶段一提到的高并发场景 Case）。
   |      4. 知识资产化：在内部工作区按照目标语言的业界标准，生成针对文件、核心类、复杂函数、难理解的核心逻辑的带规范的拟注入注释草案。
   +-- [输出] :
   |      [新增] `reports/comments_draft.md`：全量代码注释内容库草案，包含目标文件路径、目前代码内容的注释范围、注释规范与原则等内容。

                               ||
                               \/

【阶段四：产物组装与实体注入】
   |
   +-- [输入] : 阶段一的 `reports/repo_overview.md`、阶段二的 `reports/architecture.md` 和 `reports/final_tree.md`、阶段三的注释草案、增强版源码
   |
   +-- [处理] :
   |      1. 报告整合：将阶段一和阶段二的相关产物无缝融合成最终的《全方位深度解读报告》。
   |      2. 物理注释融合：将注释精准写入代码文件。
   |      3. 完整性自检：校验生成的文档是否覆盖了所有需求点，检查注入的注释是否符合预设的注释规范。
   |
   +-- [输出] :
   |	   1.[新增] `reports/final_report.md`《全方位深度解读报告》(业务价值篇 & 架构设计篇)
          2. 注入了详尽规范注释的"全新代码库"。

                               ||
                               \/

【阶段五：交付与审查 】
   +-- [输入] : 全新代码库、深度解读报告
   |
   +-- [处理] : 生成代码变更对比视图 (Diff Viewer)。
   |
   +-- [输出] :  最终确认版报告。
