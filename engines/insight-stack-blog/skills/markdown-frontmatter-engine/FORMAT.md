# Frontmatter 输出格式规范

## 基础格式

```yaml
---
<field1>: <value1>

<field2>:
  - <item1>
  - <item2>
---

<正文内容>
```

## 规则

- 字段与字段之间空一行
- 列表项缩进 2 个空格
- `---` 闭合后空一行，再接正文

## 字段顺序

字段的输出顺序由 `schema/` 目录下的文件名数字前缀决定：

```
01.title.md   -> 第 1 位
02.aliases.md -> 第 2 位
03.tags.md    -> 第 3 位
04.source.md  -> 第 4 位
05.domain.md   -> 第 5 位
```

**如何控制顺序**：在 `schema/` 下新建字段文件时，用数字前缀选择插入位置即可（如新增 `status` 字段，想排在 `tags` 之后，就命名为 `04.status.md`，原 `04.source.md` 顺延为 `05.source.md`）。无数字前缀的文件视为扩展字段，排在所有有前缀的字段之后（按字母顺序）。
