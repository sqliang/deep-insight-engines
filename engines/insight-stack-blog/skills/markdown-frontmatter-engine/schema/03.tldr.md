# tldr

- 类型：`string`
- 说明：一句话极简总结（<50字），直接抛出核心动作或结论，用于列表页的极速价值判断。

## 规则 (严格执行)
1. **字数红线**：绝对控制在 50 字以内，一句话完结。
2. **一针见血**：提炼全局最核心结论或绝对价值，拒绝机械截取拼凑正文首段。
3. **禁止废话**：绝不可使用“本文介绍了”、“主要说明了”、“作者认为”等无信息量的过渡语。

## 示例 (Few-Shot)
- 长篇前端构建工具迁移教程与性能对比
  -> ✅ "将前端工程从 Webpack 迁移至 Vite 的完整指南，冷启动提速 10 倍。"
  -> ❌ "本文详细介绍了我在项目中是如何替换构建工具的过程。" (啰嗦，无结论价值)
- 关于大模型 Agent 路由分发架构的深度笔记
  -> ✅ "提出一种基于意图识别的 Agent 路由架构，解决多模型调度时的上下文冲突。"
  -> ❌ "这篇文章探讨了 AI Agent 在处理任务时遇到的问题。" (含废话铺垫，无具体信息点)
