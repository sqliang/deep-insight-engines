# quality

- 类型：`'outstanding' | 'excellent' | 'good' | 'fair' | 'poor'`
- 说明：文章整体质量的粗粒度评级，用于快速过滤信息噪音和筛选顶尖阅读源。

## 规则 (严格执行)
1. **强制枚举**：只能从上述 5 个级别中选择，禁止生造词汇。
2. **防讨好机制（核心红线）**：默认绝大多数文章的基准线是 `good`（及格）或 `fair`（平庸）。**必须极度克制使用 `outstanding` 和 `excellent`**，拒绝评分通货膨胀。
3. **硬核评级锚点**：
   - `outstanding`（极度稀缺）：打破认知常识、提出深度的原创理论/架构、具有极强的数据支撑与行业穿透力。
   - `excellent`（高质量）：逻辑严密、结构清晰、具备极高的实操落地价值（如硬核深度教程）或深刻洞见。
   - `good`（普通基准）：信息有效、通顺完整，但属于常规经验总结或泛泛的知识搬运，缺乏惊艳的原创性。
   - `fair`（鸡肋）：内容较浅，多为观点的简单拼凑、机器翻译感重、缺乏论据支撑。
   - `poor`（垃圾）：标题党、逻辑混乱、充满废话、甚至含有明显的事实错误。

## 示例 (Few-Shot)
- 一篇开创性提出 Agent 动态反思架构并附带完整对比实验数据的深度长文
  -> ✅ `outstanding`
- 一篇排版良好、步骤清晰的常规 Vue3 组件封装教程
  -> ✅ `good` (❌ 严禁打 `excellent`，因为它只是常规知识整理，缺乏深度原创)
- 一篇通篇口号“AI 将颠覆一切”但没有任何实质案例和推导逻辑的文章
  -> ✅ `poor` (❌ 严禁因为有 AI 热点词就打高分)
