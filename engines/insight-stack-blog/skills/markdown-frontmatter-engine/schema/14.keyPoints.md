# keyPoints

- 类型：`Array<{ content: string, children?: KeyPoint[] }>` (递归树状结构)
- 说明：提取文章的逻辑骨架与论证脉络，将线性文本还原为结构化的思维导图。

## 规则 (严格执行)
1. **结构红线**：必须输出包含 `content` 和可选 `children` 的**对象数组**，绝对禁止输出纯字符串数组。
2. **正交互补**：与 `highlight` 严格区分。`highlight` 抓取散落的“闪光点”，本字段负责客观解构**全篇连贯的逻辑主干**。
3. **树状层级**：利用 `children` 构建论证树。父节点的 `content` 为章节主体或核心论点，子节点为具体的支撑论据、实施步骤或分支细节。
4. **粒度与全貌**：`content` 必须是精炼的短句（禁止长篇大论）。提取的脉络必须按原文顺序，覆盖从开篇到结尾的完整逻辑流。建议嵌套 2-3 层。

## 示例 (Few-Shot - 必须符合 YAML 嵌套规范)
- 一篇结构清晰的技术架构文章
  -> ✅
  ```yaml
  keyPoints:
    - content: "大模型长文本处理痛点"
      children:
        - content: "超长下文导致中间注意力丢失"
        - content: "算力成本呈指数级上升"
    - content: "动态块检索架构方案"
      children:
        - content: "预处理：基于语义边界的切块"
        - content: "推理期：滑动窗口注意力分发机制"
    - content: "实验数据与落地收益"
      children:
        - content: "事实召回率提升 42%"
        - content: "推理延迟降低 15%"
  ```
  -> ❌
  ```yaml
  keyPoints: ["大模型痛点", "提出解决方案", "得出实验数据"]
  ```
  (❌ 错误原因：输出成了扁平字符串数组，丢失了 `children` 支撑细节，且破坏了对象结构)
