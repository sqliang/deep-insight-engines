# domain

- 类型：`'Technology' | 'Science' | 'Humanities' | 'Business' | 'Society' | 'Personal' | 'Other'`
- 说明：提取文章所属的最顶层知识领域，用于构建知识库的宏观拓扑聚合。

## 提取规则 (强制收敛)
1. **升维归类（核心动作）**：跳出文章的具体细节特征（如“React”、“大模型”、“重构”），向上抽象识别其底层的学科或大领域归属。
2. **强制枚举（红线）**：必须且只能从上述定义的 7 个顶层领域中选择最核心的一个。绝不可自行生造词汇（❌ 绝对禁止输出 'Computer Science'、'AI'、'Programming' 等中低层分类）。
3. **领域映射基准**：
   - `Technology`：计算机科学、工程架构、软件开发、AI 算法、IT 工具。
   - `Science`：数学、物理、医学、生物等自然科学与硬核基础理论。
   - `Humanities`：哲学、历史、文学、心理学、艺术等社科与人文思潮。
   - `Business`：商业模式、创新管理、经济金融、创业方法论。
   - `Society`：政治、时事评论、社会趋势观察。
   - `Personal`：个人生活记录、游记、情绪日记、私人账单。
4. **单一归属**：若文章跨领域（例如“AI技术对商业模式的冲击”），请选择**文章核心论点侧重**的那一个，仅输出单个字符串。

## 示例 (Few-Shot)
- 文本讨论了 Vue3 组合式 API 的内存泄漏问题
  -> ✅ `domain: "Technology"` (❌ 禁止输出 "前端" 或 "编程")
- 文本分析了《思考，快与慢》中的系统1与系统2
  -> ✅ `domain: "Humanities"` (❌ 禁止输出 "心理学")
- 文本探讨了 SaaS 企业的订阅制增长飞轮
  -> ✅ `domain: "Business"`
- 文本记录了 2025 年春节回家的见闻体会
  -> ✅ `domain: "Personal"`
