# tags

- 类型：`string[]`
- 说明：基于正文内容的语义，提取描述性的标签。

## 提取规则
1. **数量约束**：严格输出 2-5 个标签。
2. **格式规范（强制）**：
   - 必须全小写。
   - 英文单词之间必须使用中划线 `-` 连接（禁止使用空格或下划线）。
   - 概念存在归属关系时，使用 `/` 表示层级。
3. **内容来源**：只允许从正文分析、提炼，**绝对禁止**参考文件名或目录路径。
4. **必带类型标签（首位优先）**：
   - 列表的第 1 个标签，必须从以下枚举池中选择最符合的一个：`[concept, tutorial, resource, thought, project, methodology, review]`。若无完全匹配的，选择最接近的。


## 示例 (Few-Shot Examples)
- ❌ 错误：`["Machine Learning", "AI 教程", "python_code"]`
- ✅ 正确：`["tutorial", "machine-learning", "python/data-processing"]`
- ❌ 错误：`["随笔", "My Project", "api设计"]`
- ✅ 正确：`["thought", "project-management", "api-design"]`
