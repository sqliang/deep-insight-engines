# title

- 类型：`string`
- 说明：代表文章核心主题的精炼标题。

## 提取规则 (按优先级执行)
1. **提取与质检（优先级 1）**：扫描正文开头是否存在一级标题（如 `# xxx`）。
   - **拦截规则（黑名单）**：如果该标题属于以下“无意义”特征之一，**必须放弃提取**，直接进入优先级 2 的生成逻辑：
     - 默认或占位符：如 `未命名`、`Untitled`、`新建笔记`、`test`、`测试` 等。
     - 纯时间/日期：如 `2023-10-12`、`周二会议` 等（除非后文有具体主题）。
     - 极短字符：去除符号后少于 3 个字符（如 `# a`）。
   - **执行提取**：如果标题有效（未触发拦截），请直接提取该文本，移除 `#` 符号，**保持原意，禁止改写**。

2. **生成兜底（优先级 2）**：如果正文没有一级标题，**或者原有标题触发了拦截规则被视为无效**，请仔细阅读全文内容，重新生成一个标题。
   - **风格**：要求言简意赅，像专业文章的标题，拒绝“震惊体”或过于口语化的表达。
   - **长度**：必须控制在 40 个字符以内。

3. **一致性**：生成的标题必须与文章主体的语种保持一致。
4. **排除项**：绝对禁止直接复制文件名作为标题。

## 示例 (Few-Shot Examples)
- 正文首行 `# 2024年 React 性能优化指南`
  -> ✅ `title: "2024年 React 性能优化指南"` (有效，直接提取)
- 正文首行 `# 临时记录`，正文内容是关于 Docker 镜像构建的报错排查
  -> ✅ `title: "Docker 镜像构建报错排查记录"` (触发拦截，基于内容重新生成)
- 正文首行 `# test`
  -> ✅ `title: "..."` (触发拦截，根据具体正文重新生成)
