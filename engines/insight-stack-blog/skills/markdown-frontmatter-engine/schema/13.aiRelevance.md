# aiRelevance

- 类型：`'high' | 'medium' | 'low'`
- 说明：评估全文与人工智能（AI/LLM/Agent等）的核心相关性，用于构建个人知识库的垂直信息过滤器。

## 规则 (严格执行)
1. **强制枚举**：只能输出上述 3 个值之一。
2. **评级基准（核心红线）**：
   - `high`（核心驱动）：文章的**绝对主体**是 AI 技术。例如：大模型底层原理、Agent 架构设计、MCP 协议解析、AI Native 应用开发。
   - `medium`（场景辅助）：AI 不是主角，而是作为辅助工具、功能特性或次要背景被深入探讨。例如：如何使用 Cursor 辅助 React 开发、某传统 SaaS 产品接入 AI 接口的商业分析。
   - `low`（无关或边缘）：文章核心与 AI 毫无关系，或者 AI 仅仅作为“蹭热点”的流行语一闪而过。例如：纯前端 MVVM 原理、日常复盘、没有任何 AI 元素的团队管理心得。
3. **宁低不高（防自嗨机制）**：评估必须极度克制。若犹豫不决，一律向下压级（即 `high` 压到 `medium`，`medium` 压到 `low`）。

## 示例 (Few-Shot)
- "Claude Code CLI 自定义 Agent 技能开发指南"
  -> ✅ `high`
- "Vue3 组合式 API 实践（附我的 ChatGPT 调优 Prompt）"
  -> ✅ `medium` (AI 只是辅助工具)
- "2025 年前端开发趋势前瞻"（文中顺嘴提了一句“AI 正在取代初级程序员”，后续全在讲工程化）
  -> ✅ `low` (❌ 严禁因为出现“AI”字眼就打 `medium` 或 `high`)
