# highlight

- 类型：`string[]`
- 说明：提炼 3-5 条最具认知冲击力、反直觉或极具传播价值的“闪光点”（与结构摘要严格正交）。

## 规则 (严格执行)
1. **数量红线**：严格输出 3-5 条，不可多也不可少。
2. **拒绝大纲（核心红线）**：绝不可写成“首先介绍了...其次分析了...”。必须是具体的、脱离上下文依然具有传播力的“独立洞见”。
3. **价值提取靶点**：只提取满足以下任一特征的“Wow 时刻”：
   - **反直觉结论**：打破常规认知或行业常识的观点。
   - **惊艳数据**：极具说服力、反差感强烈的核心数据或案例。
   - **独创模型**：作者首创的思维框架、实用技巧或犀利警示。
4. **极简独立**：每条陈述必须一语中的，直接抛出价值点。

## 示例 (Few-Shot)
- 一篇关于 SaaS 增长策略的文章
  -> ❌ `["分析了当前的行业现状", "探讨了产品驱动增长(PLG)的优势", "总结了未来的转型建议"]` (这是目录大纲，绝对禁止)
  -> ✅ `[
    "反直觉结论：在 ARR 低于 1000 万刀前，过早引入销售团队会摧毁产品自驱力，使 CAC 飙升 200%。",
    "核心洞见：最好的留存不是靠打折，而是产品内部适时触发的 Nudge（微推）机制。",
    "警示：中小企业最大的陷阱是用 '定制化服务' 去掩盖 '核心标准化能力' 的缺失。"
  ]`
