# intent

- 类型：`{ type: 'inform' | 'persuade' | 'entertain' | 'instruct' | 'summarize', description: string }`
- 说明：识别文章的核心功能意图，设定阅读期望，匹配用户当前的阅读目标（例如找教程 vs 看观点）。

## 规则 (严格执行)
1. **结构红线**：必须输出一个包含 `type` 和 `description` 两个键的对象。
2. **强制枚举 (`type`)**：只能从以下 5 项中选出最符合全文主旨的一项：
   - `instruct`: 教学指导（如：Step-by-step 教程、操作指南、源码解析）。
   - `persuade`: 观点输出与说服（如：行业趋势评判、辩论、方法论优劣对比）。
   - `summarize`: 汇总整理（如：干货清单、资源合集、每周资讯周报）。
   - `inform`: 事实告知（如：新闻发布、客观事件报道、纯概念百科）。
   - `entertain`: 休闲娱乐（如：段子、小说、非严肃的个人趣事）。
3. **精准短描 (`description`)**：用一句话（<30 字）精炼说明该意图的具体指向。拒绝废话（❌ 禁写“这篇文章的意图是...”）。

## 示例 (Few-Shot - 必须符合 YAML 对象规范)
- 一篇详细的 Vue3 性能优化指南及代码实现
  -> ✅
  ```yaml
  intent:
    type: "instruct"
    description: "提供具体的 Vue3 性能诊断与代码级优化指导"
  ```
- 标题为“为什么我不建议初创团队使用微服务”的文章
  -> ✅
  ```yaml
  intent:
    type: "persuade"
    description: "论证微服务的维护成本陷阱，试图说服读者采用单体架构"
  ```
- 某技术周报的第 152 期，包含大量链接
  -> ✅
  ```yaml
  intent:
    type: "summarize"
    description: "汇总过去一周的前端开源生态进展与优秀文章"
  ```
  -> ❌
  ```yaml
  intent: "summarize"
  ```
  (❌ 错误原因：输出成了纯字符串，丢失了对象结构和 description 字段)
